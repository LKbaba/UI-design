# 三阶段自检协议

> 什么时候读这份：build 完成后、交付前（verify 阶段）。也在有动效任务时 build 前读动效硬规则章节。

---

## 总览

```
Phase 1（结构层）→ 通过 → Phase 2（视觉层）→ 通过 → Phase 3（设计卓越）→ 通过 → 交付
         ↓ 失败                    ↓ 失败                    ↓ 失败
     修复 → 重跑 P1          修复 → 重跑 P1+P2         修复 → 重跑 P2+P3
```

**硬规则**：任何一阶段失败 → 修复 → 从该阶段重跑（P2 失败需重跑 P1+P2）。不降低标准。

---

## Phase 1 · 结构层

检查代码是否能正常运行。

### 类型检查（如用 TypeScript）

```bash
pnpm typecheck     # 或 tsc --noEmit
```

- [ ] 无 TypeScript 报错
- [ ] 无 `@ts-ignore` / `@ts-expect-error`（除非有注释说明原因）

### 控制台

- [ ] 浏览器控制台无报错（红色错误）
- [ ] 无未处理的 Promise rejection
- [ ] 无弃用警告（黄色警告可接受但应记录）

### 布局结构

- [ ] 无水平溢出（任何宽度下都不出现横向滚动条）
- [ ] 固定尺寸元素在小屏幕上不会撑破容器
- [ ] 图片/视频有明确的宽高或 aspect-ratio，不引起布局偏移

### 键盘导航与焦点样式

- [ ] 所有交互元素（按钮、链接、输入框）可用 Tab 键到达
- [ ] Tab 顺序合理（从上到下、从左到右）
- [ ] 焦点环可见（**禁止** `outline: none` 不加替代）
- [ ] 焦点环使用 `:focus-visible`（不是 `:focus`，避免鼠标点击也显示）
- [ ] 模态框有焦点陷阱（Tab 不会跳出弹窗）
- [ ] Esc 键可关闭模态框/下拉

推荐焦点样式：
```css
:focus-visible {
  outline: 2px solid var(--accent, #2563EB);
  outline-offset: 2px;
}
```

### 资源加载

- [ ] 所有图片/字体/图标能正常加载（无 404）
- [ ] 外部资源有 fallback（字体加载失败 → 系统字体）

## Phase 2 · 视觉层

检查视觉是否符合设计意图。需要截图对比。

### 截图检查

用 Playwright 或让用户在浏览器中检查以下三个宽度：

| 宽度 | 代表设备 | 重点检查 |
|------|---------|---------|
| 1440px | 桌面 | 栅格对齐、容器居中、间距节奏 |
| 1024px | 小桌面/平板横屏 | 多列网格过渡（如 4 列 → 2×2）、侧栏宽度 |
| 768px | 平板竖屏 | 布局切换点、栅格坍缩、导航切换 |
| 375px | 手机 | 触控目标、文字可读性、无溢出 |

### 对比度

| 组合 | 最低要求 |
|------|---------|
| 正文 / 主背景 | ≥ 4.5:1（WCAG AA） |
| 大号文字（≥24px bold 或 ≥18.66px bold）/ 主背景 | ≥ 3:1 |
| UI 组件（按钮、输入框边框）/ 背景 | ≥ 3:1 |
| 高对比场景（如黑底白字） | ≥ 15:1 |

验证方法：
```javascript
// 在浏览器控制台快速检查
// 安装 axe-core 或使用 Chrome DevTools Accessibility 面板
```

### 间距

- [ ] 所有间距来自 `4·8·12·16·24·32·48·64·96·128` 刻度
- [ ] 垂直节奏一致（同级 section 间距相等）
- [ ] 无"看起来差不多"的间距（要么一样，要么明确不同）

### 排版

- [ ] 正文 ≥ 16px
- [ ] 行高在推荐区间内（正文 1.5-1.6，标题 1.1-1.3）
- [ ] 只用了 2-3 个字重
- [ ] 行宽 ≤ 75 字符

### 内容

- [ ] 无编造数据（假 logo / 假评价 / 假指标）
- [ ] 占位内容用灰块 + "待替换"标注
- [ ] 图标风格统一（一套图标库）

---

## Phase 3 · 设计卓越

检查设计是否达到专业水准。

### 视觉层级

- [ ] 页面有明确的视觉焦点（用户第一眼看哪里？）
- [ ] 标题 → 正文 → 辅助信息的层级通过字号/字重/颜色区分
- [ ] CTA 按钮在视觉上最突出

### 间距节奏

- [ ] 相关元素靠近（亲密性原则）
- [ ] 不相关元素有明确间隔
- [ ] Section 间距有规律（不是忽大忽小）

### 颜色和谐

- [ ] 颜色全部来自 `DESIGN.md` 的 tokens
- [ ] 没有随手调的"差不多的颜色"
- [ ] 页面整体色彩和谐（不刺眼、不冲突）

### 情绪贴合

- [ ] 设计的整体感觉匹配 brief 描述的品牌调性
- [ ] 不会让用户觉得"这不是我们的产品"

## 动效硬规则

在 build 阶段处理动效前读这一段。

### 时长限制

| 类型 | 最大时长 | 推荐 |
|------|---------|------|
| 微交互（hover / focus） | 200ms | 150ms |
| 状态切换（展开 / 折叠） | 400ms | 300ms |
| 入场动画 | 600ms | 400ms |
| 页面转场 | 400ms | 300ms |

### 缓动函数

- ✅ `ease-out` — 入场（从快到慢，自然）
- ✅ `ease-in-out` — 状态切换
- ❌ `linear` — 除非进度条/加载圈
- ❌ `ease-in` — 出场可以用，但入场不要用（感觉迟钝）

### 硬规则

- [ ] 同屏不超过 3 个动画同时运行
- [ ] 必须尊重 `prefers-reduced-motion`：
  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      animation-iteration-count: 1 !important;
      transition-duration: 0.01ms !important;
    }
  }
  ```
- [ ] 动效服务于信息传达，不是装饰
- [ ] 不用 `animation-delay` 做错落入场（同类元素同时出现）
- [ ] 滚动触发动画不影响阅读节奏

## Playwright 使用判断

### 需要 Playwright 截图的场景

- ✅ Hero 区块动效完成后
- ✅ 新页面首次构建完成
- ✅ 深度设计评审
- ✅ 复杂交互（多状态 / 多步骤流程）
- ✅ 响应式断点验证

### 不需要 Playwright 的场景

- ❌ 纯文案修改
- ❌ 单色微调
- ❌ 已验证页面的小迭代（用户可自行刷新查看）
- ❌ 间距微调（4px → 8px）

### Playwright 截图流程

```text
1. browser_navigate → 目标页面
2. browser_take_screenshot → 全页截图（桌面宽度）
3. 分析截图，标注问题
4. 如需响应式验证 → 调整 viewport → 再次截图
5. browser_console_messages → 检查有无控制台报错
```

## 失败处理

| 失败阶段 | 处理 |
|---------|------|
| Phase 1 | 修复代码 → 重跑 Phase 1 |
| Phase 2 | 修复视觉 → 重跑 Phase 1 + Phase 2 |
| Phase 3 | 修复设计 → 重跑 Phase 2 + Phase 3 |

**绝对不做**：
- 降低验收标准（"这个对比度差一点也行吧"）
- 跳过某个阶段（"结构没问题就不看视觉了"）
- 用"用户没说要"作为不验证的理由
